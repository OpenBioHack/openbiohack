# Investigation Decision Log — [Subject ID]

> One file per investigation. Captures the significant decisions made during the work,
> so the process itself can be audited and improved over time. Significant decisions only
> — not every micro-thought. Entries are written **close to the moment** of the decision;
> reconstructed entries drift toward post-hoc justification and lose the actual reasoning.

## Conventions

- One entry per significant decision.
- Each entry timestamped (date + approximate time).
- Tagged with the step the decision occurred in (e.g., `[Step 3 — Routes]`).
- Newest entry at the top.
- Brief and concrete — what was decided, what the alternatives were, what tipped it.

## What counts as significant — log these

- A candidate pathway entering the working set, or being set aside (and why).
- A research question accepted as discriminating, or rejected as not.
- An interview-loop exit for a specific candidate (interview-saturated → discriminator stage), with the evidence that ended interviewing for it.
- A tier change for any candidate, and what triggered it.
- A halt-and-reassess (what was discovered, what was re-audited, what was changed).
- A sub-agent conclusion accepted as-stated, or downgraded one tier on relay.
- A synthesis / connector candidate passing or failing the hollow-test gate.
- A choice between competing models when the evidence didn't fully resolve it (provisional commitment).
- A discriminator chosen as a trial vs a test, with the trade-off named.

## Entry template

```
### YYYY-MM-DD HH:MM — [Step X — Step Name] — [Short decision title]

- **Context:** the state of the investigation when this came up
- **Decision:** what was decided
- **Alternatives considered:** the other options on the table
- **What tipped it:** the specific evidence or reasoning that moved the call
- **Tier impact:** which candidates moved (up/down/added/parked) as a result
- **Next:** what this enables or what it now requires
```
